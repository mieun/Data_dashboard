## 응답 규칙

- 한국어로 설명한다
- 데이터는 data/ 폴더의 CSV만 사용한다

## 분석 규칙

- 거래금액 음수·결측·중복은 제거한다
- 차트는 한글 폰트(Malgun Gothic)를 적용한다
- 결과 차트는 05_이미지자료 폴더에 저장한다

# 프롬프트 ① 데이터 진단

03_실습데이터/01_핀테크결제_dirty.csv 를 읽고, 결측치·중복·이상치가
각각 어디에 얼마나 있는지 표로 정리해서 알려줘.

# 프롬프트 ② 전처리 (번호로 구체적으로!)

위 파일을 다음 순서로 정제해줘.

1) 완전 중복 행 제거
2) '연령대','지역','결제수단' 결측치는 최빈값으로 채우기
3) 거래금액이 비었거나 음수인 행 제거
4) '결제수단'의 앞뒤 공백 제거
5) 거래일시 날짜 형식 통일

결과를 03_실습데이터/핀테크_정제완료.csv 로 저장하고,
처리 전후 행 수와 남은 결측치 개수를 알려줘.

# 프롬프트 ③ 검증

정제된 파일의 거래금액 통계(describe)와 결제수단 value_counts를
보여줘. 이상한 값이 남아있는지 점검해줘.

# 프롬프트 ④ 차트 생성

핀테크_정제완료.csv 로 아래 3개 차트를 한글 깨짐 없이 만들어 PNG로 저장해줘.

- 업종별 총 거래금액 막대그래프 (파란색, 값 라벨 표시)
- 결제수단 비중 도넛차트
- 월별 거래액 추이 선그래프

# 프롬프트 ⑤ 대시보드 생성

이 데이터로 Streamlit 대시보드를 만들어줘.

- 사이드바: 지역·업종 필터
- 상단: 총 거래액·건수·평균·고객수 KPI 카드
- 본문: 업종별 막대 + 결제수단 도넛 + 월별 추이

streamlit run 으로 실행까지 해줘.

# 프롬프트 ⑥ 수정 요청 (검증 후 다듬기)

막대그래프 색을 파란색(#2563EB)으로 바꾸고,
각 막대 위에 값을 표시해줘.
